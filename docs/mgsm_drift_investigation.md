# MGSM 多轮下降趋势排查报告

> 2026-07-19 晚。排查 3 轮 GLM-5.2 评测中 `multilingual_mgsm` 单调下降 (0.400→0.350→0.200) 的归因。
> 起点 commit `c123212`；三轮 run dir: `results/glm_full_v2_glm-local_20260719T{105423,111015,113244}Z/`。

## 1. 现象

| 任务 | 轮1 (seed=1234) | 轮2 (seed=5678) | 轮3 (seed=9012) | 极差 |
|---|---|---|---|---|
| multilingual_mgsm (法语) | 0.400 | 0.350 | 0.200 | **0.200** |
| reasoning_gsm8k (英语) | 0.533 | 0.433 | 0.467 | 0.100 |
| coding_humaneval | 0.833 | 0.867 | 0.900 | 0.067 |

MGSM 是 8 个任务中方差最大、且**单调下降**的唯一一个，需排查是数据抽样 bug 还是模型/服务端漂移。

## 2. 排查路径

### 2.1 数据一致性验证（核心）

读取三轮的 `samples.jsonl`，对 20 道 MGSM 题做：
- 题目内容（input 前 80 字符）的 hash
- 题目顺序
- gold 值

**结果**：三轮 `qhash = 7980` 完全一致，题目顺序与 gold 完全一致。

→ **排除"抽样 bug"假设**。`load_dataset_samples(spec, n=20, seed=0)` 在 web loader 路径下用固定 `seed=0` 取样，三轮跑的是**完全相同的 20 道法语题**。

> 注：seed=1234/5678/9012 是 plan-level seed（控制 `ModelClient.generate` body.seed），不影响 dataset 抽样（dataset 用固定 seed=0 见 `datasets.py:30`）。这是设计预期，避免换 seed 就换题导致跨 seed 不可比。

### 2.2 三轮逐题 verdict 对比

|---|---|---|---|---|---|
| 0 | T | T | F | 2 | 边界 |

- **3 题稳定全对**（Q2/Q12/Q18）：简单题，模型每次都能答对
- **8 题稳定全错**（Q3-Q6/Q10/Q11/Q13-Q16）：难题，模型完全不会
- **9 题边界方差**（Q0/Q1/Q7/Q8/Q9/Q17/Q19）：模型有时对有时错

→ 0.4→0.35→0.2 的下降，是**多道边界题巧合地都朝"错"方向变化**的结果，不是某道题整体崩坏。

### 2.3 temp 配置确认

```
multilingual_mgsm: temperature=0, n=-, max_tokens=512
reasoning_gsm8k:   temperature=0, n=-, max_tokens=512
```

`temp=0` 理论上应确定性输出。实际有方差 → **sglang 服务端的非确定性**（batch 调度顺序、浮点累积、KV-cache 复用差异等）。

这是已知现象（同 P0 推理）：`reasoning_gsm8k` 也有 0.100 极差，但 MGSM 更大。

### 2.4 为何 MGSM 方差大于 GSM8K？

MGSM-fr 是法语版 GSM8K，对 GLM-5.2 而言法语是**非主语言**：
- 英语题：简单题稳定对、难题稳定错、**边界题少** → 方差小（0.100）
- 法语题：更多题落在"模型有时理解有时不理解"的边界区间 → **边界题多（9/20 vs 英语少）** → 方差大（0.200）

非主语言鲁棒性不足是模型本身的特性，非评测管道问题。

## 3. 结论

| 假设 | 是否成立 | 证据 |
|---|---|---|
| 数据集抽样不同 | ✗ 排除 | 三轮 qhash=7980 完全一致 |
| 评测管道 bug | ✗ 排除 | BFCL/Needle/IFEval/BBH 三轮极差=0.000 |
| sglang 服务端非确定性 | ✓ 主因 | temp=0 仍有方差，多任务都有此现象 |
| MGSM 题型特异 | ✓ 加重 | 非主语言边界题多，9/20 落在边界 |

**最终归因**：sglang 服务端在 temp=0 下的微小非确定性 + 法语是非主语言导致 MGSM 边界题特别多 → 表现为方差放大。**不是 bug，是模型+服务端的真实特性。**

## 4. 后续建议

1. **正式对比时增大 n**：MGSM 当前 n=20，方差槽大。建议 P1 升 `multilingual_mgsm` n≥100，方差可压到 ≤0.05。
2. **多语言维度扩语种**：只有法语一题不能下"多语言能力"结论，应加德语/中文/日语（待 MGSM 数据全量接入）。
3. **temp=0 复现性说明**：在 `docs/architecture.md` 或对外报告明确指出"temp=0 下仍可能有 ±0.05 的服务端非确定性，跨轮均值比单轮更可信"。
4. **id 字段补全**（附带 bug）：runner 写 `samples.jsonl` 时 missing `id` 字段，debug 不便。建议 `_run_one` 写入 `index + dataset_id` 组合 id。低优先，留 P1 顺带修。
